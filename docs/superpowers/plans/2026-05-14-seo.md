# SEO Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add client-side SEO to Gen Con Buddy: per-route meta tags, Open Graph, Schema.org structured data on event pages, `robots.txt`, and `sitemap.xml`.

**Architecture:** TanStack Router's built-in `head()` option per route sets dynamic titles and meta tags; `<HeadContent />` renders them to the DOM via React 19's native element hoisting. `index.html` carries a full static fallback block for social crawlers that don't run JS. Event detail pages additionally emit Schema.org JSON-LD structured data via the `'script:ld+json'` key in the `meta` array. The existing `usePageTitle` hook is deleted — title ownership moves to `head()`.

**Tech Stack:** TanStack Router v1.114 (`HeadContent`, `head()` route option), React 19 (native `<title>`/`<meta>` hoisting), Vitest + JSDOM (test via `document.querySelector`)

import rootStyles from "./__root.module.css";

const POSTHOG_TOKEN = import.meta.env.VITE_PUBLIC_POSTHOG_PROJECT_TOKEN as string | undefined;

function PageViewTracker(): null {
  const posthog = usePostHog();
  const { location } = useRouterState();

  useEffect(() => {
    posthog.capture("$pageview");
  }, [location.href, posthog]);

  return null;
}

const SITE_DESCRIPTION =
  "Fast, deeply filterable event search for Gen Con. Search across event type, time, location, cost, and more.";

function AppShell(): React.JSX.Element {
  const { theme, setTheme } = useTheme();

  return (
    <div className={indexStyles.page}>
      <HeadContent />
      {POSTHOG_TOKEN && <PageViewTracker />}
      <header role="banner" className={indexStyles.header}>
        <Link to="/" className={rootStyles.brandingTitle}>
          <Meeple3D size={32} aria-hidden="true" />
          Gen Con Buddy
        </Link>
        <p className={rootStyles.tagline}>your guide to the best four days in gaming</p>
        <nav className={rootStyles.nav}>
          <Link to="/" activeOptions={{ exact: true, includeSearch: false }}>
            Search
          </Link>
          <Link
            to="/changelog"
            search={{
              open: [],
              eventType: undefined,
              days: undefined,
              timeStart: undefined,
              timeEnd: undefined,
            }}
            activeOptions={{ includeSearch: false }}
          >
            Changelog
          </Link>
          <ThemePopover theme={theme} setTheme={setTheme} />
        </nav>
        <div className={rootStyles.mobileNav}>
          <MobileNav theme={theme} setTheme={setTheme} />
        </div>
      </header>
      <Outlet />
      <footer className={rootStyles.footer}>
        <Link to="/about" className={rootStyles.footerLink}>
          About
        </Link>
      </footer>
    </div>
  );
}

export interface RouterContext {
  queryClient: QueryClient;
}

export const Route = createRootRouteWithContext<RouterContext>()({
  head: () => ({
    meta: [
      { title: "Gen Con Buddy" },
      { name: "description", content: SITE_DESCRIPTION },
      { property: "og:type", content: "website" },
      { property: "og:site_name", content: "Gen Con Buddy" },
      { property: "og:title", content: "Gen Con Buddy" },
      { property: "og:description", content: SITE_DESCRIPTION },
      { property: "og:image", content: "https://gcb.quest/og-image.png" },
      { property: "og:image:width", content: "1200" },
      { property: "og:image:height", content: "630" },
      { name: "twitter:card", content: "summary_large_image" },
    ],
  }),
  component: (): React.JSX.Element =>
    POSTHOG_TOKEN ? (

          ui_host: (import.meta.env.VITE_PUBLIC_POSTHOG_HOST as string) || "https://us.posthog.com",
          defaults: "2026-01-30",
          capture_exceptions: true,
          debug: import.meta.env.DEV,

    ) : (

    ),
});

### Step 3: Create placeholder og-image.png

The `og-image.png` needs to be a 1200×630 PNG. Use any of the following:

**Option A — Node.js one-liner (requires `sharp` installed globally or via npx):**

```bash
npx sharp-cli --input public/favicon.svg --output public/og-image.png resize 1200 630
```

**Option B — ImageMagick:**

```bash
convert -size 1200x630 xc:"#f5f0e8" -font DejaVu-Sans -pointsize 72 -fill "#3d2b1f" \
  -gravity Center -annotate 0 "Gen Con Buddy" public/og-image.png
```

**Option C — Minimal valid PNG (pure white 1×1, works to prevent 404, replace with real design later):**

```bash
node -e "
const { createCanvas } = require('canvas');
// If canvas not available, copy favicon.svg as a fallback
" 2>/dev/null || cp public/favicon.svg /tmp/og.svg
```

If none of these are available, create a 1200×630 image using any image editor (Figma, GIMP, etc.) and save as `public/og-image.png`. A placeholder solid-color image is fine for now. The key requirement is that `https://gcb.quest/og-image.png` does not 404.

## Problem

When a user shares an event URL (e.g. \`https://gcb.quest/event/RPG26123456\`) on Discord, Slack, or Twitter, the platform's link preview shows the generic Gen Con Buddy homepage description instead of the event's name, type, and details.

This happens because social platform crawlers do not execute JavaScript. They see only the static \`index.html\` shell, which carries sitewide fallback og: tags.

## Solution

The API (already running on Railway) needs bot-detection middleware:

1. Inspect the incoming \`User-Agent\` header for known social crawlers (Discordbot, Twitterbot, Slackbot, facebookexternalhit, LinkedInBot, etc.).
2. If matched: fetch the event record from the database using the \`gameId\` path parameter, then return a minimal HTML document with fully-populated \`og:title\`, \`og:description\`, and \`og:image\` tags. The body can be empty — crawlers only read the \`<head>\`.
3. If not matched: serve the normal Vite SPA \`index.html\` as today.

This requires no SSR of the React app — just a thin route handler in the existing Express/Fastify API that handles \`/event/:id\` for bot user agents.

## Acceptance criteria

- Sharing \`https://gcb.quest/event/RPG26123456\` on Discord shows the event title and a one-line description
- Normal browser navigation to the same URL is unaffected
- The solution does not require migrating the frontend to SSR
EOF
)"
